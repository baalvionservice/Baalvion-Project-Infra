# Support

## How to get help

- **Documentation** — start with [`README.md`](./README.md), the per-domain
  `Backend/services/<domain>/README.md`, and the [`docs/`](./docs) directory
  (architecture, ADRs, runbooks).
- **Questions & discussion** — open a [Discussion](../../discussions) or ask in
  the team's internal channel.
- **Bugs & feature requests** — open a [GitHub Issue](../../issues/new/choose)
  using the appropriate template.
- **Security vulnerabilities** — **do not** open a public issue. Follow
  [SECURITY.md](./SECURITY.md) and email security@baalvion.com.
- **Operational / production incidents** — page the on-call rotation via the
  internal incident process; do not rely on GitHub issues for live incidents.

## Before filing an issue

1. Search existing issues to avoid duplicates.
2. Confirm you are on the latest `main`.
3. Include the affected service/app, commit SHA, environment, and clear
   reproduction steps.
